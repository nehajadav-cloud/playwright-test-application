Features

  • Role-based auth: admin and viewer
  • Employees list: search, filter, sort, pagination
  • Create, edit, delete, bulk actions (admin only)
  • Employee detail page with edit/delete
  • QA toggles: latency, forced failure, failure rate
  • Rich test hooks via data-testid

Navigation

  • / Login page
  • /employees Employees list
  • /employees/:id Employee detail page
  • /docs This documentation

QA Controls

These controls let you simulate slow or failing APIs in UI tests.

  • Delay: adds latency to all /api calls
  • Fail: forces a 503 response
  • Fail rate: random failures (0-1)
You can also set headers manually in tests: 
x-qa-delay: 1000
x-qa-fail: 1
x-qa-fail-rate: 0.3

API

Auth

  • POST /api/login body: { username, password, remember }
  • POST /api/logout
  • GET /api/me

Employees

  • GET /api/employees?q=&status=&sortBy=&sortDir=&page=&pageSize=
  • GET /api/employees/:id
  • POST /api/employees (admin)
  • PUT /api/employees/:id (admin)
  • DELETE /api/employees/:id (admin)
  • POST /api/employees/bulk (admin)

Architecture

  • Node/Express backend serves API and static HTML
  • In-memory sessions with cookie-based auth
  • Employee data stored in backend/data/employees.json
  • Static UI pages in backend/public

Files

  • backend/server.js Express server and API routes
  • backend/public/login.html Login UI
  • backend/public/employees.html Employees list UI
  • backend/public/employee.html Employee detail UI
  • backend/public/docs.html Documentation (this page)

Playwright Ideas

  • Login success and failure flows
  • Role restriction: viewer cannot create/edit/delete
  • Pagination and sorting assertions
  • Search and filter accuracy
  • Bulk actions with confirmation dialogs
  • Failure simulation and retries
